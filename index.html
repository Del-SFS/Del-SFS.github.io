SCAN

SFS Comprehensive Archive Network

Overview

SCAN (SFS Comprehensive Archive Network) is a structured and automated mod management system for the PC version of Spaceflight Simulator. Its primary goal is to replace manual installation workflows with a deterministic and reproducible system.

Traditional mod installation requires users to manually extract files and place them into specific directories. This process is error-prone and inconsistent, especially when dealing with multiple mods, dependencies, or conflicting assets. SCAN eliminates these issues by enforcing a standardized format and automating the entire installation pipeline.

Interface

The user interface of SCAN is designed to expose only the necessary controls while abstracting file system complexity. Users interact with mods as structured entries rather than raw files.

This abstraction allows SCAN to maintain consistency and ensure that every installation follows the same rules.

Core System

At its core, SCAN operates on structured metadata defined in JSON format. Each mod entry contains essential information such as version, type, dependencies, and compatibility constraints.

During installation, SCAN parses this metadata and determines the correct destination paths for each file. This eliminates ambiguity and ensures that all assets are placed correctly within the game directory.

  • Deterministic installation logic
  • Automatic directory mapping
  • Structured metadata parsing
  • Consistent mod handling

Mod Packaging Standard

SCAN enforces a strict packaging format to ensure compatibility across all mods. Every mod must follow the same structural pattern. 

ModName.zip
 ├── ModName.zip
 └── mod.json

The inner archive must contain only the raw mod files (DLLs, assets, textures, etc). Nested directories are not allowed as they introduce ambiguity in file placement.

Dependency and Conflict Handling

SCAN is designed to support complex mod environments where multiple mods interact. Each mod can declare dependencies and conflicts, allowing SCAN to evaluate compatibility before installation.

This prevents scenarios where incompatible mods overwrite each other or break the game state.

  • Dependency resolution system
  • Conflict detection before install
  • Pre-install validation

Automation Pipeline

SCAN is part of a broader ecosystem that includes tools for mod creation and repository management.

The pipeline consists of:

  • GMJS → Generates valid mod.json files
  • ModCenterSFS → Stores structured mod data
  • SCAN → Installs and manages mods

This pipeline ensures that all components operate using the same standards, reducing inconsistencies.

Installation Process

To use SCAN, simply download the application and execute it. The system will automatically prepare the environment and allow mod installation.

No manual file manipulation is required, and all operations are handled internally by the application.

Project Objective

The long-term goal of SCAN is to create a fully standardized and automated ecosystem for mod distribution.

By enforcing strict formats and automating installation, SCAN enables scalability and reliability for both developers and users.
