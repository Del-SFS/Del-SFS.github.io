SCAN

SFS Comprehensive Archive Network

Overview

SCAN is a structured mod management system designed for the PC version of Spaceflight Simulator. It replaces manual installation workflows with a deterministic and automated process.

Instead of relying on users to manually place files into the correct directories, SCAN interprets structured metadata and performs all operations programmatically. This ensures consistency across installations and eliminates common configuration errors.

System Architecture

The SCAN ecosystem is based on a data-driven architecture. All mod operations are derived from structured JSON metadata, allowing the application to remain predictable and extensible.

Each mod entry defines its own installation rules, dependencies, and compatibility constraints. SCAN evaluates this data before executing any file operations.

  • Metadata-driven installation
  • Deterministic execution model
  • Separation between data and logic

Installation Model

SCAN maps mod components to predefined directories based on their type. This removes ambiguity and guarantees that files are always placed in the correct location.

  • DLL → Mods directory
  • Parts → Custom Assets/Parts
  • Textures → Texture Packs
  • Systems → Custom Solar Systems

This mapping is enforced programmatically and cannot be altered by user error.

Mod Packaging Standard

To ensure compatibility, SCAN enforces a strict packaging format. All mods must follow the same structural pattern. 

ModName.zip
 ├── ModName.zip
 └── mod.json

The internal archive must contain only raw mod files. Nested directories introduce ambiguity and are not allowed.

Dependency and Conflict Resolution

SCAN supports complex mod environments by allowing each mod to declare dependencies and conflicts.

Before installation, SCAN evaluates these relationships to prevent incompatible configurations.

  • Pre-install validation
  • Dependency enforcement
  • Conflict prevention

Automation Pipeline

SCAN is part of a unified ecosystem designed to standardize mod development and distribution.

  • GMJS → Generates structured mod definitions
  • ModCenterSFS → Stores mod metadata
  • SCAN → Executes installation and management

This pipeline ensures that all components operate under the same rules and formats.

Execution Flow

When SCAN runs, it performs the following sequence:

  • Load repository metadata
  • Parse mod definitions
  • Validate dependencies and conflicts
  • Determine file mappings
  • Execute installation

Each step is deterministic, ensuring reproducible results across different systems.

Objective

The objective of SCAN is to create a reliable and scalable infrastructure for mod distribution.

By enforcing strict standards and automating installation, SCAN reduces fragmentation and improves the overall modding experience.
